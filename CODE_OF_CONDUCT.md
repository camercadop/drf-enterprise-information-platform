# Code of Conduct

## Our Commitment

We are committed to providing a welcoming, inclusive, and respectful environment for all contributors, regardless of background, identity, or experience level. This platform is an enterprise-grade Django REST Framework project designed for professional use in multi-tenant environments.

## Expected Behavior

As a contributor to this project, you are expected to:

- Use welcoming and inclusive language
- Respect differing viewpoints and experiences
- Gracefully accept constructive criticism
- Focus on what is best for the community
- Show empathy towards other community members
- Be mindful of your words and actions
- Demonstrate professionalism in all interactions

## Unacceptable Behavior

Unacceptable behaviors include:

- The use of sexualized language or imagery
- Personal attacks, insults, or derogatory comments
- Trolling or public shaming
- Discrimination based on protected characteristics
- Inappropriate comments related to project security
- Unwelcome sexual attention or advances
- Publishing private information without consent
- Inappropriate technical content or demonstrations
- Any behavior that could reasonably be considered inappropriate in a professional setting

## Reporting Issues

If you experience unacceptable behavior, please report it by contacting the project maintainers:

- GitHub repository: https://github.com/camercadop/drf-enterprise-information-platform
- For security-related issues, see our security documentation

All reports will be reviewed and investigated promptly. The project maintainers will respond to all reports within a reasonable timeframe.

## Attribution

This Code of Conduct is adapted from the [Contributor Covenant](https://www.contributor-covenant.org/), version 2.1, available at https://www.contributor-covenant.org/code-of-conduct.html.

## Project-Specific Guidelines

Given this is an enterprise platform with multi-tenant architecture:

- Security discussions should be conducted through proper channels
- Production-related changes require thorough testing and review
- Tenant isolation concerns should be reported as security issues
- All contributions should follow the project's [development guidelines](docs/development.md)

## Enforcement

Instances of abusive, harassing, or otherwise unacceptable behavior may be reported to the project maintainers who will take appropriate action. The project maintainers reserve the right to remove, edit, or reject contributions that do not align with this Code of Conduct.
